libstorage: Design Concepts and Main Overview

Short overview about the concepts and classes of libstorage. Click on the class names to get a more detailed description of each class.

class Storage

This is the main class with that one can get access to the functionality provided by libstorage. The class contains a list of container objects.

All modifying member functions of the storage library will go through Storage class. This is the central place where things like readonly access, locking, testmode, inst-sys etc. are handled. So e.g. if one wants to remove a partition from a certain disk he will call Storage::RemovePartition(...) not Disk::RemovePartition(...). Of course Storage::RemovePartition(...) may eventually call Disk::RemovePartition(...) but only after the checks against the central restrictions (e.g. readonly access or active testmode) have succeeded in class Storage.

This has the additional advantage the the complete class hierarchy below Storage could be changed without affecting the user interface of libstorage.

class Container

This is the base class for everything being able to contain volumes. A Container contains a list of Volumes. Currently, the following classes derived from Container exist: Disk, LvmVg, Evms.

class Disk

Derived class from Container. Contains knowledge about handling of normal disks.

class LvmVg

Derived class from Container. Contains knowledge about handling of LVM Volume groups.

class Evms

Derived class from Container. Contains knowledge about handling of EVMS Entities.

class Volume

This is the base class for all objects that can be formatted and mounted. It contains cunctionality for formatting, mounting, fstab handling. Currently, the following classes derived from Volume exist: Partition, LvmLv, EvmsVol, Md, Loop.

class Partition

Derived class from Volume. Contains data about a disk partition.

class LvmLv

Derived class from Volume. Contains data about a LVM Logical Volume.

class EvmsVol

Derived class from Volume. Contains data about a EVMS Volume.

class Md

Derived class from Volume. Contains data about a Software Raid Device.

class Loop

Derived class from Volume. Contains data about a file based loop filesystem.

class AsciiFile

Class for accessing and modifying ascii files.

class ProcPart

Derived class from AsciiFile. Class for accessing content of /proc/partitions.
